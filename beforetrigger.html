TRIGGER IS A NAMED PL/SQL PROGRAM UNIT ASSOCIATED WITH A SPECIFIC DATABASE TABLE. IT IS STORED IN THE DATABASE AND EXECUTED AUTOMATICALLY IN RESPONSE TO A SPECIFIC EVENT THAT OCCURS IN THE TABLE.


BEFORE TRIGGER:
WHEN DEFINING A TRIGGER, YOU CAN SPECIFY THE TRIGGER TIMING. THAT IS, YOU CAN SPECIFY WHETHER THE TRIGGER ACTION IS TO BE EXECUTED BEFORE OR AFTER THE TRIGGERING STATEMENT.


CREATE OR REPLACE TRIGGER tr
        BEFORE INSERT ON stu
        REFERENCING OLD AS pre NEW AS post
        FOR EACH ROW
        WHEN (post.roll > 10)
        BEGIN
            IF INSERTING THEN
                :POST.name := UPPER(:POST.name);
            END IF;
        END;
        
    
        Trigger created.

        insert into stu values(47,'aman');
        select * from stu;

AFTER TRIGGER:
AFTER TRIGGERS ARE FIRED AFTER THE TRIGGERING ACTION SI COMPLETED.

create or replace trigger tr 
            after delete on stu 
                referencing old as pre new as post 
            for each row  
            begin 
            if deleting then 
            dbms_output.put_line(:pre.name); 
            end if; 
            end; 
        
            delete from stu where roll=8;
        
            1 row(s) deleted.
            nitin
